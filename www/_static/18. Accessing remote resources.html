Accessing Remote Resources

Web pages and data

I have mentioned before how one can access data files on your hard drive, but Python also allows you to access remote data, for example on the internet. The easiest way to do this is to use the built-in urllib module. To start off, you have to open the URL and keep it as a Python object:

In [1]: 
from urllib import urlopen
u = urlopen('http://xkcd.com/353/')

Note that in Python 3, you should instead use:

from urllib.request import urlopen

Now, u will behave similarly to a (read-only) file object. You can read the contents as you would a normal file:

In [2]: 
content = u.read()
In [3]: 
print content[:1000]  # only print the first 1000 characters

APIs

Imagine that you want to access some data online. In some cases, you will need to download a web page and search through the HTML to extract what you want. For example:

In [8]: 
u = urlopen('http://www.wetteronline.de/wetter/heidelberg')
content = u.read()

This is not ideal because it is messy, and also slow if all you want are a couple of values. A number of websites now offer an "Application programming interface" (or API) which is basically a way of accessing data is a machine-readable way. Let's take a look at http://openweathermap.org/ for example, which has an API: http://openweathermap.org/API. To access the weather for Heidelberg, you can do:

In [10]: 
u = urlopen('http://api.openweathermap.org/data/2.5/weather?q=Heidelberg,Germany')
content = u.read()
